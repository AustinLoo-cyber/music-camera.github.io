# 🎵 Music Camera 🎵
The Music Camera (a.k.a. Music Portrait) is a web application that imagines how one would look if ones favourite songs were shown on their physical self.

## 🚀 Key Features
- **Camera Utilities**: The application uses MediaPipe's camera utilities to access the user's camera and provide interactive features.
- **Pose Detection**: The project utilizes MediaPipe's pose detection library to track the user's movements and gestures.
- **Custom Typography**: The application features custom fonts, including Bookmania, to provide a unique and visually appealing design.
- **Modern Design**: The project's CSS styles define a modern and dark color scheme, with a noise texture overlay, creating a cohesive and engaging user experience.

## 🛠️ Tech Stack
- **Frontend**: HTML, CSS, JavaScript
- **Libraries**: MediaPipe (camera utilities, selfie segmentation, and pose detection)
- **Fonts**: Google Fonts (Syne and DM Mono), Custom Fonts (Bookmania)
- **Frameworks**: None

## 📦 Getting Started / Setup Instructions
### If you want to use the website
1. Go to the given website: https://austinloo-cyber.github.io/music-camera/
2. If you want to test how the features work, click on 'Use Demo Tracks'.
3. Configure how you want to see the songs in the settings below.
4. Then, click 'Start Camera'.
5. If you're using your phone, you can switch between front and back camera.

### If you want to run locally
1. Clone the repository using `git clone https://github.com/your-repo/music-portrait.git`
2. Just click on index.html for it to open on your local browser.
3. If there is camera blocking issue, do try and host it at local server and open it under 'localhost:8080' or whatever localhost url that is given.


## 📱 Current status
Tests on this website has only been done on demo tracks and manually typing in your favourite songs. The Spotify and Youtube Music methods have not been fully tested. Do use them at your own risk.
